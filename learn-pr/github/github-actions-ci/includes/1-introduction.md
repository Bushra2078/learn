*Continuous integration* (CI) is the process of automating a software development team's build and test tasks so that they can be done quickly and often. This process helps elevate the team's performance by catching issues early allowing for more agility.

In this module, you learn how to implement continuous integration using GitHub Actions and GitHub Workflows in your GitHub repositories.

## Learning objectives

In this module, you:

- Build a GitHub Workflow from a template to build and test a Node.js project
- Debug a failed test using the GitHub Actions Log
- Customize your workflow to:
  - create a build artifact and save it in GitHub Artifacts
  - get access to your build artifacts
  - test against multiple targets
  - add labels your pull requests using a GitHub action
