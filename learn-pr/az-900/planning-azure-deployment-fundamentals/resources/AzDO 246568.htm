AFUN20A: Azure Core Architectural Components Fundamentals

Learn Module Azure Core Architectural Components Fundamentals
MS Learn Learning Path LP1: Fundamental Cloud Concepts and Core Azure Services
ELP module mapping AFUN 20a
Abstract Tailwind Traders needs to understand how and where they should deploy their Azure workloads as they migrate to and adopt Azure. At present they are not fully sure about how they can deploy in such a way that they follow best practices as well as ensuring that they meet compliance requirements around high availability.
AZ 900 Objectives Covered
  • Describe the core azure architectural components
    • Describe the benefits and usage of Regions and Region Pairs
    • Describe the benefits and usage of Availability Zones
    • Describe the benefits and usage of Resource Groups
    • Describe the benefits and usage of Subscriptions
    • Describe the benefits and usage of Management Groups
    • Explain Azure resources
    • Describe the benefit and usage of Azure Resource Manager
        • Not: ARM templates, which is covered in 3.2, or RBAC which is in 5.1
Units
  • Azure Regions and Region Pairs
  • Azure Availability Zones
  • Azure Management Groups and Subscriptions
  • Azure Resources
  • Azure Resource Groups
  • Benefits of Azure Resource Manager
  • Case Study Knowledge Check
  • Summary & Further Learning
Problems addressed / Case Study Question Ideas
  • When should I use separate resource groups and when should I use separate subscriptions?
  • How can I ensure my workload is highly available?
  • How do I know which region to choose in my own country? Do I just look at a map? Is there some way to figure out which region is the best for my organization or are they all the same?
  • How do I ensure that my organization’s data and workloads remain in a specific geographic region?
  • I have IaaS VMs, Azure SQL Instances, container workloads and App Service apps. When should I segment workloads by resource group?
  • When does it make sense to have more than one subscription?
  • What are the drawbacks of using multiple subscriptions?
  • What is the relationship between subscriptions and Azure AD tenancies?
  • What does ARM allow me to do that I can’t do using a script?
  • ARM templates look complicated, is there an easy way for me to understand them or do I even need to?
  • What are some drawbacks to ARM?
Demonstrations / Potential Sandbox Exercises
  • ARM template introduction video
  • Show a workload that is deployed across availability zones and regions (maybe Azure Site Recovery for VMs across regions)
  • Create and delete resource groups
  • Show permissions inheritance
  • Collect subscriptions into a management group
  • Deploy a VM from a GitHub ARM template
Notes
  • Addresses what attendees need to know about Azure architecture and structure in terms of how and where they deploy workloads.
