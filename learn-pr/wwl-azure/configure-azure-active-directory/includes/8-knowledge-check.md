## Multiple Choice
Users want to sign-in to devices, apps, and services from anywhere. They want to sign-in using an organizational work or school account instead of a personal account. IT requires that corporate assets are protected and that devices meet standards for security and compliance. What should be done? 
( ) Add the device in Azure Active Directory {{That's incorrect. To get a device under the control of Azure AD either join or register the device. }}
(x) Join the device to Azure Active Directory {{That's correct. Azure AD device registration provides the device with an identity that is used to authenticate the device when a user signs-in to Azure AD. The identity can be used to enable or disable a device. }}
( ) Register a device in Azure Active Directory. {{That's incorrect. Joining a device is an extension to registering a device. Joining provides the benefits of registering and changes the local state of a device. Changing the local state enables users to sign-in to a device using an organizational work or school account instead of a personal account. }}
