## Multiple Choice
Virtual network peering is not successfully established until the peering status for both virtual network peerings shows status?
(x) Connected {{That's correct. The peering is not successfully established until the peering status for both virtual network peerings is Connected.}}
( ) Initiated {{That's incorrect. The Initiated status is shown when the peering is first created.}}
( ) Peered {{That's incorrect. Peered is not a virtual network status. }}

## Multiple Choice
Which of the following allows peered virtual networks to share the gateway and get access to resources?
(x) Gateway transit {{That's correct. Gateway transit allows peered virtual networks to share the gateway and get access to resources.}}
( ) Point-to-site connectivity {{That's incorrect. Point-to-site creates a secure connection to a virtual network from an individual client computer.}}
( ) Transitivity {{That's incorrect. Transitivity refers to virtual networks communicating with the peers of their peers.}}

## Multiple Choice
Which of the following is true about virtual network peering?
( ) Creating a virtual network disrupts other resources. {{That's incorrect. There is no downtime to resources in either virtual network when creating the peering, or after the peering is created.}}
(x) Traffic between the virtual networks is kept on the Microsoft backbone network. {{That's correct.}}
( ) Peered virtual networks must be in the same region. {{That's incorrect. Global virtual network peering connects virtual networks that are in different Azure regions.}}
