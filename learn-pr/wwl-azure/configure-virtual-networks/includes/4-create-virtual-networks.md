You can create new virtual networks at any time. You can also add virtual networks when you create a virtual machine. Either way you will need to define the address space, and at least one subnet.

> [!NOTE]
> Default limits on Azure networking resources can change periodically so it's a good idea to consult the documentation for the latest information.

:::image type="content" source="../media/create-virtual-networks-b4f1fd40.png" alt-text="Screenshot of the create virtual network page. The project details include subscription, resource group, name and region.":::


> [!NOTE]
> Plan to use an address space that is not already in use in your organization, either on-premises or in the cloud. Even if you plan for cloud-only virtual networks, you may later decide to connect an on-premises site.
