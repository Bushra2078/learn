Model deployment is usually not part of the data scientist role. Model deployment can involve many things but it always depends on how the business is looking to implement the model. Let's outline a couple examples of what that may entail.

Using our example, this part of the process could be as simple as just choosing to implement the results of the model by stocking the shelves accordingly. It could also be as complex as building an application or integrating results into a website using HTML code.

Model deployment is the final stage of the data science procedure and is often done by a developer. Deployment typically means putting the resulting model into an environment where it can be called by applications, i.e. consumed. There are many considerations for how a model is deployed.
