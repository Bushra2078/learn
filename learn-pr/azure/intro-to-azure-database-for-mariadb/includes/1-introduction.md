Your developers  and cloud architects don't want to have to worry about  managing the server that hosts their apps. They won't want to concern themselves with the underlying database and operating system. Instead, they'll want to focus on the development of their apps. By using Azure Database for MariaDB, you can help them accelerate the deployment of their apps.

Azure Database for MariaDB is a managed relational database as a service (DBaaS) offering that can handle your mission-critical workloads with predictable performance and dynamic scalability. You can easily connect your existing client apps and database tools to your instance of Azure Database for MariaDB to get the benefits of a Platform as a Service (PaaS) environment.

:::image type="content" source="../media/mariadb-conceptual-diagram.png" alt-text="A graphic depicting the architecture of Azure Database for MariaDB in a typical organization.":::

By choosing to implement Azure Database for MariaDB, you can provide for: 

- High-availability
- Data protection
- Predictable and scalable performance

> [!NOTE]
> Azure Database for MariaDB is based on the MariaDB community edition database engine version 10.2 and 10.3.

## Example scenario

If your organization currently implements MariaDB databases to support critical workloads, you could benefit from migrating to Azure Database for MariaDB. Azure Database for MariaDB provides for:

- Built-in high-availability
- Automatic scaling
- Automatic backups and point-in-time restore
- Excellent compliance and security 

This means your cloud architects and developers can focus on app development, and delivery of those apps and their data to your users.

In this module, you'll identify the features of Azure Database for MariaDB. You'll be able to assess whether Azure Database for MariaDB can help you address your organization's database requirements. You'll contrast Azure Database for MariaDB with MariaDB on Azure VMs.

By the end of this module, you'll be able to evaluate whether Azure Database for MariaDB is appropriate to handle your organization's mission-critical workloads.
