Providing rich documentation for an API makes it more consumable and easier to maintain. In the case of your framing business, you used Swashbuckle to auto-generate Swagger documentation for your ASP.NET Core API. Just adding Swashbuckle packages to your project provides you professional Swagger documentation that is easy to read and doesn't require the consumer to know the source code of the API. you then learned how to enhance the default documentation through XML comments and annotations.
