As you are preparing to teach this module, get familiar with what the students will learn during the module. In this module, students will explore the characteristics of non-relational data. Students will also learn about the different types of non-relational data and non-relational database management systems. 

This module consists of three lessons: 

- **Lesson 1** – Explore non-relational data offerings in Azure 

- **Lesson 2** – Explore provisioning and deploying non-relational data services in Azure 

- **Lesson 3** – Manage non-relational data stores in Azure 

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4XRBR] 

### Module 3 Tips and Tricks 

- Explain the difference between storage account and storage options within the account 

- Utilize demos on Lesson 2 for each storage type 

  - If time allows cover content before in class demo 

- Explain what makes Azure Cosmos database different from other database services 

  - Discuss any limitations that students may see when using a relational database to collect information 

- Consider creating an Azure Cosmos database account as prep before instruction 

  - Utilize the DP-900 resource group 

  - Consider utilizing the data migration tool to import volcano [sample data](https://aka.ms/azurecosmosdbsampledata) into the cosmos account 

- Focus on explaining the consistency in a globally distributed database and its importance. 

  - Utilize [this documentation](https://aka.ms/consistencylevelsinazurecosmosdb) when explaining consistency level in Azure Cosmos database
